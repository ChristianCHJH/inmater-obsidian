# Índice: Templates de Documentación

## 📋 Archivos en esta carpeta

| Archivo | Descripción | Propósito | Tipo |
|---------|-------------|----------|------|
| [trace-template.md](trace-template.md) | Template para traces (investigaciones) | Documentar flujos completos: UI → API → DB | Trace |
| [adr-template.md](adr-template.md) | Template para ADRs (decisiones arquitectónicas) | Documentar decisiones de diseño y opciones consideradas | ADR |
| [domain-model-template.md](domain-model-template.md) | Template para modelos de dominio (DDD) | Documentar conceptos, agregados, reglas de negocio | Domain |
| [endpoint-doc-template.md](endpoint-doc-template.md) | Template para documentación de endpoints | Documentar endpoints REST: request, response, validaciones | Endpoint |

## 📖 Cómo usar

Cuando crees un nuevo documento:

1. **Para una investigación (trace):** Copia [trace-template.md](trace-template.md)
2. **Para una decisión arquitectónica:** Copia [adr-template.md](adr-template.md)
3. **Para documentar un dominio:** Copia [domain-model-template.md](domain-model-template.md)
4. **Para documentar un endpoint:** Copia [endpoint-doc-template.md](endpoint-doc-template.md)

Cada template contiene instrucciones inline de qué completar.
